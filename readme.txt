1. Creational Patterns (Object Creation)
1.1 Singleton Pattern - Thread safe , prevents multiple instances
1.2. Factory Pattern  - Abstracts object creation, improves maintainability

2. Structural Patterns (Object Composition)
2.1 Adapter Pattern - Useful for integrating old APIs with new systems.
2.2 Decorator Pattern - Adds features dynamically instead of modifying base structs
2.3 Proxy Pattern - Used for security , caching and lazy loading.

3. Behavioral Patterns (Object Interaction)
3.1 Stratedy Pattern - Useful for switching between differnet implementations dynamically.
3.2 Observer Pattern - Useful for event-driven syatems like messaging, stock markets, etc.
3.3 Command Pattern  - Great for undo/redo operations in applications.
